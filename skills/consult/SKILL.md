argument-hint: <preflight-json> <review-decision-input-json> <consultation-request-json>
allowed-tools: Read, Grep, Glob, Write, Agent, Bash(rigor consult-decide *), Bash(rigor consult-start *), Bash(rigor consult-finish *)

Run `rigor consult-decide --input <review-decision-input-json>` before requesting Codex. The input must copy validated risk, assessment, failure-progress, concern, transmission, and observed plugin-availability facts; do not change them to force a request. Continue to Codex only for `request-independent-review` with `invocationAllowed: true`. Honor `skip-independent-review`, `continue-claude-only`, and the nonzero `stop-required-review` exactly as returned. Transmission denial is evaluated first and always forbids invocation, even for high risk or an explicit human request. Unknown plugin availability is not availability.

Then run `rigor consult-start --preflight <preflight-json> --input <consultation-request-json>` immediately before invoking Codex. This is the hard external-transmission prerequisite; stop if it exits nonzero even when the advisory decision requested review. Send only the requested decision, the minimum relevant repository-relative diff and files, the contract constraints, and necessary verification facts to `codex:codex-rescue`. Do not send the whole Claude transcript or unnecessary repository content. For review or adversarial-review mode, keep the task read-only; prompt constraints are advisory, so the deterministic finish check remains mandatory.

Ask Codex to separate facts, options, recommendation, uncertainty, and human decisions. Do not request hidden reasoning or chain of thought. Invoke it synchronously unless the user explicitly requests background execution and the job can still be finalized in this session.

After the result, write a minimal `rigor.consultation-result-input.v2` JSON file outside the repository. Normalize every finding to bounded severity, repository-relative evidence path and optional line, reproducibility, required action, and confidence. Set unavailable usage, model, and reasoning-effort statuses explicitly to `"unavailable"`, omit the corresponding values, and never infer model, effort, tokens, or cost. Do not store the raw transcript or chain of thought. Run `rigor consult-finish --session <saved-session-path> --input <result-json>`. If it reports `mutated-worktree`, stop and show the unexpected path/state change. A zero-finding result and Codex agreement are advisory observations, never verification, approval, or merge permission.
